# Renderer Adapter

## Purpose
`render_with_drawio_adapter.py` takes the prompt files generated by `run_architecture_skill.py` and executes a configurable renderer command to create final `.png` and `.drawio` outputs.

## Why the adapter is configurable
The exact command to invoke drawio-skill can vary by environment. For that reason, the adapter uses a configurable shell command template.

## Placeholders
The command template supports these placeholders:
- `{prompt}` — prompt markdown file path
- `{png}` — target PNG file path
- `{drawio}` — target Draw.io XML / `.drawio` file path
- `{name}` — logical job name
- `{run_dir}` — current run directory

## Example config
See `examples/renderer_config.json`.

Example command template:
```text
YOUR_RENDER_COMMAND --prompt-file "{prompt}" --output-png "{png}" --output-drawio "{drawio}"
```

## Usage
### Dry run
```bash
python render_with_drawio_adapter.py   --run-dir outputs/<run_name>   --config examples/renderer_config.json   --dry-run
```

### Execute rendering
```bash
python render_with_drawio_adapter.py   --run-dir outputs/<run_name>   --command-template 'YOUR_RENDER_COMMAND --prompt-file "{prompt}" --output-png "{png}" --output-drawio "{drawio}"'   --allow-parallel
```

### Force sequential rendering
```bash
python render_with_drawio_adapter.py   --run-dir outputs/<run_name>   --config examples/renderer_config.json   --force-sequential
```

## Parallel fallback
The adapter probes whether the execution engine supports parallel execution.
If not, or if a parallel stage fails, it falls back to sequential execution.

## Output
The adapter writes:
- `render_report.json`
- final `.png` files
- final `.drawio` files
